## 装饰者模式
包装器(Wrapper)

#### 意图Intent
动态地给一个对象添加一些额外的职责, 装饰器为子类在扩展功能时, 提供了灵活且可选择的方案.

**装饰者模式思想**：以具体饮料(继承Beverage基类)为主体，然后在运行时以调料来“装饰”饮料。

例如客户需要摩卡和奶泡深焙咖啡，那么要做的是：.

* 1、拿一个深焙咖啡对象；
* 2、以摩卡对象装饰；
* 3、以奶泡对象装饰；

摩卡和奶泡属于调料，但是也是装饰者，它的类型反映了它装饰的对象，所谓反映，指的是两者类型一致。那么所有调料需要继承Beverage。
